# Account AI for Tally — notes for Claude

An MCP (Model Context Protocol) server that connects Claude and other AI assistants to **Tally Prime** accounting software. Written in TypeScript (Node 22+) and compiled with **Bun** into a single cross-platform binary. It talks to Tally over its HTTP/XML gateway (UTF-16), and caches report data in an in-memory SQLite database (`sql.js`, WebAssembly — no native dependencies).

## Structure

- `code/` — the npm package (all source and build config)
  - `src/mcp.mts` — registers every MCP tool (reports, `query-database`, master + voucher tools)
  - `src/tally.mts` — Tally HTTP transport + the "pull" (report/export) engine
  - `src/push.mts` — the "push" (import) builders: create/update/delete masters, vouchers
  - `src/database.mts` — the sql.js in-memory cache (`cacheTable` / `executeSQL`)
  - `src/config.mts` — `~/.account-ai-for-tally/config.json` (host/port/company), read live
  - `src/main.mts` — entry dispatcher: `--mcp` → MCP stdio; no args → control panel
  - `src/control-panel.mts` — local setup web UI (127.0.0.1:4321) + its JSON API
  - `src/claude-config.mts` — registers the binary with Claude Desktop / Claude Code
  - `src/index.mts`, `src/mcp-stdio.mts` — the stdio MCP entry; `src/server.mts` — remote-HTTP/OAuth variant (hosted-path foundation, not the primary flow)
  - `pull/` — report XML/TDL templates (source of truth; inlined into the binary at build)
  - `scripts/build-embeds.mjs` — inlines `pull/*` + the sql.js wasm into `src/generated/` (gitignored, regenerated by the build)
- `documentation/` — CHANGELOG, design docs (`distribution-plan.md`, `control-panel-plan.md`), and the Tally XML reference
- `makefile` — task runner (see below)

Build artifacts (`code/dist/`, `code/dist-bin/`, `code/src/generated/`, `node_modules/`) are gitignored.

## Commands

Run from the repo root:

- `make setup` — install dependencies
- `make build` — compile to `code/dist` (Node)
- `make test` — run the test suite (Vitest)
- `make typecheck` — `tsc --noEmit`
- `make binary` / `make binaries` — build the single binary for this OS / all OSes (needs Bun)
- `make panel` / `make mcp` — run the control panel / the MCP server locally
- `make clean` — remove build artifacts

## Rules

- **Before committing, lint and test the affected files.** Run `make typecheck` and `make test`, and lint the files you changed with `npx biome check <files>` (from `code/`). Do not commit if typecheck or tests fail. The repo is not fully Biome-clean at baseline, so only lint your own changed files — do not reformat the whole tree.
